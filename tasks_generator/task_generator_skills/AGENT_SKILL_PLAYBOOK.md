# Agent Skill Playbook（通用版）

用于指导 Agent：从已有 task 出发，分析并产出可复用的 generator。

## 核心流程（5 步）

### 1) Analyze（分析原始任务）

- 明确任务目标与完成标准。
- 明确评分关注点（结果正确性 / 过程完整性 / 安全约束）。
- 明确依赖边界（数据、服务、工具、环境）。

### 2) Abstract（抽象生成模型）

- 提取 **不变量**：必须始终成立的规则。
- 提取 **变量**：可变化且能产生多样性的维度。
- 定义 **失败边界**：哪些行为应直接失败或显著扣分。

### 3) Generate（实现生成器）

- 先做最小可运行样本，再扩展批量生成。
- 为每个样本输出可追溯 metadata（如 seed、配置、关键答案信息）。
- 保持任务产物自包含，避免隐式依赖。

### 4) Validate（验证一致性）

- 结构一致性：文件完整、路径稳定。
- 运行一致性：可加载、可执行、可评分。
- 语义一致性：任务描述、评分规则、参考行为三者一致。

### 5) Iterate（迭代优化）

- 固定 seed 重现问题。
- 收敛变体空间定位根因。
- 修复后做小样本回归 + 批量抽检。

## 最小发布检查清单

- [ ] Generator 可稳定批量产出任务
- [ ] 任务产物可独立运行
- [ ] 评分规则稳定且可解释
- [ ] 可通过 metadata 快速回溯问题
- [ ] 文档说明清楚输入参数与输出结构

## 一句话原则

先保证正确与可复现，再追求多样性与复杂度。
